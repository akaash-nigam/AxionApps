MAJOR BREAKTHROUGH: DISCOVERED CORRECT BUILD PATTERN
==========================================

**KEY DISCOVERY**: xcodebuild can work with Package.swift directly!

The correct build pattern for Package.swift apps is:
```bash
xcodebuild -scheme <SchemeName> \
  -sdk xrsimulator \
  -destination 'platform=visionOS Simulator,id=<SimulatorID>' \
  -skipPackagePluginValidation \
  -skipMacroValidation \
  build
```

To find the scheme name:
```bash
xcodebuild -list 2>&1 | grep -A 2 "Schemes:"
```

To find available simulators:
```bash
xcodebuild -list 2>&1 | grep -A 20 "Available destinations"
```

This pattern works WITHOUT needing .xcodeproj files!

==========================================
TECHNICAL INSIGHTS
==========================================

**xcodebuild with Package.swift**:
- xcodebuild resolves Package.swift as a workspace
- Uses Package.swift to generate temporary Xcode project
- Builds in .swiftpm/xcode directory
- No need for explicit .xcodeproj file

**Common Build Patterns**:
1. Apps with .xcodeproj: Use xcodebuild directly
2. Apps with Package.swift: Use xcodebuild with -scheme
3. Apps without either: Cannot build (need configuration)

**Error Patterns Observed**:
- Apps with <5 errors: Usually fixable
- Apps with 10-30 errors: Moderate effort, possible cascading
- Apps with >50 errors: Likely incomplete implementation
