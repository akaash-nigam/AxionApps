## ðŸ“ Documentation Standards

### Writing Style

All documentation follows these standards:

**Tone:**
- Clear and concise
- Professional but approachable
- Action-oriented
- Inclusive language

**Structure:**
- Table of contents for documents > 500 lines
- Consistent heading hierarchy (H1 â†’ H2 â†’ H3)
- Code examples in fenced code blocks with language tags
- Visual diagrams where appropriate (ASCII art or Mermaid)

**Formatting:**
- Markdown (GitHub-flavored)
- 80-120 character line length (soft limit)
- Consistent bullet styles (- for unordered, 1. for ordered)
- Emoji usage for visual scanning (sparingly)

### Code Examples

All code examples should:
- Include language specification: ` ```swift `
- Show complete, runnable examples when possible
- Include comments for complex logic
- Follow Swift style guide conventions
- Demonstrate best practices

Example:
```swift
// MARK: - Punch Detection

/// Detects punch from hand motion data
/// - Parameters:
///   - hand: Current hand anchor
///   - previousPosition: Last known hand position
///   - deltaTime: Time since last update
/// - Returns: Detected punch or nil
func detectPunch(
    hand: HandAnchor,
    previousPosition: SIMD3<Float>,
    deltaTime: TimeInterval
) -> Punch? {
    let velocity = calculateVelocity(hand.position, previousPosition, deltaTime)
    guard velocity.magnitude >= velocityThreshold else { return nil }
    return classifyPunch(velocity: velocity, hand: hand)
}
```
